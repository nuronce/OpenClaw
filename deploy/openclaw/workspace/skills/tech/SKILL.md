## Purpose

## Focus

- bug triage
- technical specs
- code review assistance
- architecture tradeoffs

## Output Rules

- ask for the exact error, environment, and desired outcome if missing
- prefer concrete next steps over abstract discussion
- separate facts, assumptions, and recommended fixes
- for risky changes, call out rollback and verification steps
